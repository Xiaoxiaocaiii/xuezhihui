
### 1.2 文件加载依赖关系

前端页面的依赖加载遵循以下层级结构：

```
页面HTML
├── css/main.css          ← 所有页面均加载（全局样式）

└── js/api.js             ← 需要调用后端接口的页面均加载（提供API封装）
    └── 依赖 utils.js 中的函数：

```

**各页面具体加载的JS文件**：

| 页面 | utils.js | api.js | 说明 |

| login.html | ✅ | ✅ | 需要调用login接口 + Toast提示 |
| register.html | ✅ | ✅ | 需要调用register接口 + Toast提示 |

### 2.1 设计令牌模块（行1-45）

```css
/* ============================================================
```

**`:root`**：CSS伪类选择器，匹配文档根元素（即`<html>`）。在此定义的CSS自定义属性（变量）全局可用，所有后代元素均可通过`var()`函数引用。

#### 颜色令牌（行9-18）

```css
  --color-primary:       #4f46e5;
```

**`--color-primary`**：主色调，靛蓝色（Indigo 600）。用于按钮背景、链接颜色、导航栏品牌标识等核心交互元素。`#4f46e5`在Tailwind CSS调色板中对应indigo-600，具有良好的视觉对比度和可访问性。

```css
  --color-primary-hover: #4338ca;
```

**`--color-primary-hover`**：主色调悬停态，深靛蓝（Indigo 700）。当鼠标悬停在主色调按钮上时显示，提供视觉反馈，`#4338ca`比primary深约1个色阶。

```css
  --color-primary-light: #eef2ff;
```

**`--color-primary-light`**：主色调浅色背景（Indigo 50）。用于上传区域的悬停背景色、文件预览条背景色等，与主色调形成层次对比。

```css
  --color-success: #10b981;
  --color-warning: #f59e0b;
  --color-error:   #ef4444;
  --color-info:    #3b82f6;
```

**四种语义颜色**：
- `--color-success: #10b981`（翠绿色）：成功Toast背景、成功状态标识
- `--color-warning: #f59e0b`（琥珀色）：警告Toast背景
- `--color-error: #ef4444`（红色）：错误Toast背景、危险按钮背景、输入框错误状态
- `--color-info: #3b82f6`（蓝色）：信息Toast背景

```css
  --color-white:          #ffffff;
  --color-bg:             #f8fafc;
  --color-border:         #e2e8f0;
  --color-text:           #1e293b;
  --color-text-secondary: #64748b;
  --color-text-muted:     #94a3b8;
```

**中性色令牌**：
- `--color-white: #ffffff`：纯白色，卡片和输入框背景色
- `--color-bg: #f8fafc`（Slate 50）：全局页面背景色，浅灰调减少视觉疲劳
- `--color-border: #e2e8f0`（Slate 200）：卡片边框、输入框边框、导航栏底边框
- `--color-text: #1e293b`（Slate 800）：正文文字颜色，深灰色比纯黑更柔和
- `--color-text-secondary: #64748b`（Slate 500）：次要文字（副标题、辅助信息）
- `--color-text-muted: #94a3b8`（Slate 400）：最浅文字（占位符、提示文本、文件名标签）

#### 字体令牌（行21-23）

```css
  --font-family: -apple-system, BlinkMacSystemFont, "Segoe UI",
                 "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
                 "Helvetica Neue", Arial, sans-serif;
```

**`--font-family`**：系统字体栈。优先级从高到低：
1. `-apple-system`：macOS/iOS系统默认字体（San Francisco）
2. `BlinkMacSystemFont`：macOS Chrome的系统字体
3. `"Segoe UI"`：Windows系统默认字体
4. `"PingFang SC"`：macOS中文苹方字体（简体中文）
5. `"Hiragino Sans GB"`：macOS中文冬青黑体
6. `"Microsoft YaHei"`：Windows中文微软雅黑
7. `"Helvetica Neue", Arial, sans-serif`：通用无衬线后备字体

这种设计确保在不同操作系统上均使用最佳的中英文系统字体，无需加载Web字体文件。

```css
  --font-size-xs:   0.75rem;    /* 12px — 文件名标签、上传提示 */
  --font-size-sm:   0.875rem;   /* 14px — 按钮文字、表单标签、辅助文本 */
  --font-size-base: 1rem;       /* 16px — 正文基准字号 */
  --font-size-lg:   1.125rem;   /* 18px — 标语文字 */
  --font-size-xl:   1.25rem;    /* 20px — 导航品牌名 */
  --font-size-2xl:  1.5rem;     /* 24px — 页面标题、H2 */
  --font-size-3xl:  1.875rem;   /* 30px — 着陆页大标题、H1 */
```

**基于`rem`的字号系统**：`1rem = 16px`（浏览器默认根字号）。使用`rem`而非`px`，使得字号可以随浏览器设置缩放，提升可访问性。

### 2.2 全局重置模块（行50-73）


- `box-sizing: border-box;`：将元素的`width/height`计算方式改为包含`padding`和`border`，更符合直觉的盒模型

```css
html { font-size: 16px; }
```

**根字号锁定**：明确设置`<html>`的字号为16px，避免某些浏览器默认字号不一致导致`rem`计算偏差。

```css
body {
  font-family: var(--font-family);
  font-size: var(--font-size-base);
  line-height: 1.6;
  color: var(--color-text);
  background-color: var(--color-bg);
  min-height: 100vh;
}
```

**body全局样式**：
- `font-family`：引用系统字体栈令牌
- `font-size`：16px基准字号
- `line-height: 1.6`：行高1.6倍，适中的阅读密度
- `color`：深灰色正文
- `background-color`：浅灰背景色
- `min-height: 100vh`：确保页面至少占满整个视口高度，即使内容很少也不会出现白色断层

```css
a { color: var(--color-primary); text-decoration: none; }

```

**链接样式**：默认移除下划线，使用主色调；悬停时加深颜色并显示下划线，提供视觉反馈。

```css

h1 { font-size: var(--font-size-3xl); }
h2 { font-size: var(--font-size-2xl); }
h3 { font-size: var(--font-size-xl); }
```

**标题标签统一规范**：所有标题使用`font-weight: 600`（半粗体）、紧凑行高1.3、正文颜色。

```css
img { max-width: 100%; display: block; }
```

**图片重置**：`max-width: 100%`防止图片超出容器宽度；`display: block`消除图片底部的默认间隙。

```css
ul, ol { list-style: none; }
```

**列表重置**：移除默认的项目符号/编号样式。

### 2.3 按钮系统模块（行78-102）

```css
/* ============================================================
 * 3. 按钮
 * ============================================================ */
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: var(--spacing-sm);
  padding: 0.625rem 1.25rem;

  color: var(--color-white);

  border-radius: var(--radius-md);

```

**`.btn`基础按钮类**：
- `display: inline-flex`：弹性布局，使按钮内容（图标+文字）水平居中排列
- `align-items: center`：垂直居中
- `justify-content: center`：水平居中
- `gap: var(--spacing-sm)`：按钮内多个元素（如Spinner+文字）之间的间距
- `padding: 0.625rem 1.25rem`：上下10px，左右20px的内边距
- `font: 500 var(--font-size-sm)/1.5 var(--font-family)`：简写属性，依次为字重500、14px字号、1.5行高、系统字体
- `color: var(--color-white)`：白色文字
- `background-color: var(--color-primary)`：靛蓝色背景
- `border: 2px solid var(--color-primary)`：2px实线边框，与背景同色
- `border-radius: var(--radius-md)`：8px圆角
- `cursor: pointer`：鼠标悬停时显示手型光标
- `white-space: nowrap`：按钮文字不换行

```css
.btn:hover       { background-color: var(--color-primary-hover); border-color: var(--color-primary-hover); }
.btn:focus-visible { outline: 2px solid var(--color-primary); outline-offset: 2px; }
.btn:disabled    { opacity: 0.5; cursor: not-allowed; }
```

**按钮状态**：
- `:hover`：悬停时背景和边框变为深靛蓝
- `:focus-visible`：键盘聚焦时显示2px外框线（无障碍访问），`outline-offset: 2px`使轮廓与按钮边缘有2px间距
- `:disabled`：禁用状态半透明+禁止光标

```css
.btn-outline       { color: var(--color-primary); background-color: transparent; border-color: var(--color-primary); }
.btn-outline:hover { color: var(--color-white); background-color: var(--color-primary); }
```

**`.btn-outline`轮廓按钮**：透明背景+靛蓝边框+靛蓝文字；悬停时反转为靛蓝背景+白色文字。

```css
.btn-danger        { background-color: var(--color-error); border-color: var(--color-error); }
.btn-danger:hover  { background-color: #dc2626; border-color: #dc2626; }
```

**`.btn-danger`危险按钮**：红色背景，用于删除操作；悬停时变为更深的红色(#dc2626, red-600)。

```css
.btn-sm            { padding: 0.375rem 0.75rem; font-size: var(--font-size-xs); border-radius: var(--radius-sm); }
.btn-lg            { padding: 0.75rem 1.75rem; font-size: var(--font-size-base); border-radius: var(--radius-lg); }
.btn-block         { display: flex; width: 100%; }
```

**按钮尺寸变体**：
- `.btn-sm`：小尺寸按钮（上下6px、左右12px、12px字号、6px圆角），用于导航栏
- `.btn-lg`：大尺寸按钮（上下12px、左右28px、16px字号、12px圆角），用于认证页主按钮和上传按钮

### 4.1 核心请求函数（行1-14）

**IIFE + 严格模式**：
- `(function () { ... })();`：定义后立即执行的匿名函数，创建私有作用域，避免内部变量污染全局命名空间
- `'use strict'`：启用ES5严格模式，禁止使用未声明的变量、禁止删除变量等，提升代码安全性和可维护性

```javascript

```

**`BASE_URL`**：后端API的基础URL。该变量在IIFE内部定义，不属于全局作用域，只有IIFE内部的函数可以访问。如果需要更改后端地址，只需修改此变量。

- **值**：`'http://localhost:8080'` — 指向本地运行的Spring Boot服务
- **设计考量**：硬编码在JS文件中，适合开发和学习场景；生产环境应使用相对路径或环境变量

```javascript
  async function request(method, url, data, isFormData) {
    var headers = {};
    if (!isFormData) headers['Content-Type'] = 'application/json';
    var body = data ? (isFormData ? data : JSON.stringify(data)) : null;
    var response = await fetch(BASE_URL + url, { method: method, headers: headers, body: body });
    var result = await response.json();
    if (result.code !== 200) throw new Error(result.message || '操作失败');
    return result;
  }
```

**`request(method, url, data, isFormData)`** — 核心请求函数，所有API调用都通过此函数完成：

**参数解析**：
- `method`：HTTP方法字符串（`'GET'`、`'POST'`、`'DELETE'`）
- `url`：接口路径（如`'/api/auth/login'`），与`BASE_URL`拼接形成完整URL
- `data`：请求数据（普通请求为JS对象，文件上传为FormData对象，GET请求为`null`）
- `isFormData`：布尔值，`true`表示`data`是FormData（文件上传），此时不设置`Content-Type`（让浏览器自动设置`multipart/form-data`边界）

**执行流程逐行解析**：

**第1步——构建请求头**：
```javascript
var headers = {};
if (!isFormData) headers['Content-Type'] = 'application/json';
```
- 创建空headers对象
- 如果不是FormData请求，设置`Content-Type: application/json`
- 对于FormData请求（文件上传），不设置Content-Type，让浏览器自动设置为`multipart/form-data; boundary=...`

**第2步——处理请求体**：
```javascript
var body = data ? (isFormData ? data : JSON.stringify(data)) : null;
```
- 如果`data`存在：
  - 如果是FormData（文件上传）：直接使用FormData对象作为body
  - 否则：将JS对象序列化为JSON字符串
- 如果`data`为空（如GET请求）：body为`null`

**第3步——发送HTTP请求**：
```javascript
var response = await fetch(BASE_URL + url, { method: method, headers: headers, body: body });
```
- 使用`fetch` API发送异步HTTP请求
- `await`暂停函数执行直到请求完成，返回`Response`对象
- URL由`BASE_URL`和`url`拼接，例如`http://localhost:8080/api/auth/login`

**第4步——解析响应**：
```javascript
var result = await response.json();
if (result.code !== 200) throw new Error(result.message || '操作失败');
return result;
```
- 从响应中解析JSON数据
- 检查后端返回的`code`字段：
  - `200`：成功，返回完整的result对象（包含`code`、`message`、`data`三层）
  - 非`200`：失败，抛出异常（异常消息取`result.message`，兜底为`'操作失败'`）
- 调用方使用`try/catch`捕获异常，调用`showToast()`显示错误消息

```javascript
  window.downloadResourceToLocal = async function (id) {
    var response = await fetch(BASE_URL + '/api/resources/' + id + '/download');
    var disposition = response.headers.get('Content-Disposition');
    var fileName = 'download';
    if (disposition) {
      var starMatch = disposition.match(/filename\*=UTF-8''([^;\n]*)/i);
      if (starMatch && starMatch[1]) {
        fileName = decodeURIComponent(starMatch[1].trim());
      } else {
        var match = disposition.match(/filename[^;=\n]*=((['"]).*?\2|[^;\n]*)/);
        if (match && match[1]) fileName = match[1].replace(/['"]/g, '');
      }
    }
    var blob = await response.blob();
    var url = window.URL.createObjectURL(blob);
    var a = document.createElement('a');
    a.href = url;
    a.download = fileName;
    document.body.appendChild(a);
    a.click();
    document.body.removeChild(a);
    window.URL.revokeObjectURL(url);
  };
```

**文件名解析逻辑详解**：

**第1步——获取Content-Disposition响应头**：
```javascript

```

**第2步——优先匹配RFC 5987标准编码**（`filename*=UTF-8''xxx`）：
```javascript

}
```

- `decodeURIComponent()`将百分号编码解码为UTF-8中文

**第3步——回退匹配传统格式**（`filename="xxx"`）：
```javascript
else {
    var match = disposition.match(/filename[^;=\n]*=((['"]).*?\2|[^;\n]*)/);
    if (match && match[1]) fileName = match[1].replace(/['"]/g, '');
}
```
- 正则匹配`filename="xxx"`或`filename=xxx`两种格式
- `.replace(/['"]/g, '')`去除可能存在的引号

**第4步——触发浏览器下载**：
```javascript
var blob = await response.blob();                              // 将响应体转为Blob对象
var url = window.URL.createObjectURL(blob);                     // 创建临时Blob URL
var a = document.createElement('a');                           // 创建隐藏的<a>标签
a.href = url;                                                  // 设置href为Blob URL
a.download = fileName;                                         // 设置下载文件名
document.body.appendChild(a);                                  // 临时添加到DOM（必须，否则Firefox无法触发下载）
a.click();                                                     // 模拟点击，触发下载
document.body.removeChild(a);                                  // 从DOM移除
window.URL.revokeObjectURL(url);                               // 释放Blob URL内存
```

**文件选择事件绑定**：

`files`是`<input type="file">`的FileList对象，`files[0]`获取用户选择的第一个文件。

```javascript

    e.preventDefault();
    U.zone.classList.add('dragover');
  });
  U.zone.addEventListener('dragleave', function (e) {
    e.preventDefault();
    U.zone.classList.remove('dragover');

    e.preventDefault();
    U.zone.classList.remove('dragover');

```

**拖拽事件三部曲**：
1. `dragover`：文件拖入区域上方时触发。`e.preventDefault()`是必须的（否则浏览器会默认以新页面方式打开文件）。添加`.dragover`类改变视觉样式
2. `dragleave`：文件拖离区域时触发。移除`.dragover`类
3. `drop`：文件释放时触发。从`e.dataTransfer.files`获取拖放的文件，调用`selectFile()`

`e.stopPropagation()`防止点击事件冒泡到上传区域（否则会重新触发文件选择）。

```
            │
            ▼
[api.js → request('DELETE', '/api/resources/1', {userId: 1})]
  └── fetch → DELETE http://localhost:8080/api/resources/1
       │  Body: {"userId": 1}
       │
       ▼
[ResourceController.delete(@PathVariable Integer id, @RequestBody Map body)]
  ├── resourceMapper.selectById(1)  → Resource{..., userId=1}
  ├── resource.getUserId().equals(1) → true ✓ (权限通过)
  └── resourceMapper.deleteById(1)
       │
       ▼
[ResourceMapper.xml → DELETE FROM resource WHERE id = 1]
  └── resource表删除id=1的记录
       │
       ▼
[返回: {code: 200, message: "操作成功", data: "删除成功"}]
  └── showToast('删除成功！', 'success')
  └── reload() → 重新请求 GET /api/resources 刷新列表
```

### 6.6 发表评论全链路
