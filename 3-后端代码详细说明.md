### 1.2 分层架构说明

项目采用经典的**三层架构（无Service层）**：

```
┌─────────────────────────────────────────────────┐
│                  Controller 层                    │
│  AuthController / UserController                 │
│  ResourceController / CommentController          │
│  • 接收HTTP请求，参数校验                           │
│  • 调用Mapper接口执行业务操作                        │
│  • 封装ResultUtil统一返回格式                       │
├─────────────────────────────────────────────────┤
│                    Mapper 层                      │
│  UserMapper / ResourceMapper / CommentMapper     │
│  • Java接口定义数据访问方法                          │
│  • XML文件编写实际SQL语句                           │
│  • 方法签名与XML中的SQL ID一一对应                   │
├─────────────────────────────────────────────────┤
│                    Entity 层                      │
│  User / Resource / Comment                       │
│  • 纯数据载体（POJO），使用Lombok @Data             │
│  • 字段名通过下划线转驼峰映射到数据库列名              │
│  • 同时作为API的请求/响应DTO使用                     │
└─────────────────────────────────────────────────┘
```

**架构特点**：
- Controller直接调用Mapper，省略了传统的Service层。原因是项目业务逻辑简单（基本为单表CRUD），不需要额外的业务编排层
- Entity同时承担了DTO的角色，直接用于API的请求参数和响应数据
- MyBatis使用XML映射方式（而非注解），SQL与Java代码分离，便于SQL调优和维护

**`<modelVersion>4.0.0</modelVersion>`**：POM模型版本，Maven 2/3/4均使用4.0.0，是固定的标准写法。

### 2.2 父项目依赖


**`<parent>`**：声明父POM为Spring Boot 4.0.6。继承父POM的好处：
- 统一管理所有Spring Boot相关依赖的版本号（无需在每个dependency中写version）
- 预设Maven插件配置（maven-compiler-plugin、spring-boot-maven-plugin等）
- 预设资源过滤、测试框架配置
- `<relativePath/>`为空表示从Maven远程仓库拉取父POM（而非本地相对路径）

**Maven坐标（GAV）**：
- `groupId`：组织标识，`com.xuezhihui`遵循Java包命名规范

- `name`和`description`：项目可读名称和描述

## 三、启动类 XuezhihuiApplication.java 逐行解析

**文件路径**：`src/main/java/com/xuezhihui/XuezhihuiApplication.java`

```java
package com.xuezhihui;
```

**`package com.xuezhihui`**：根包声明。Spring Boot默认从启动类所在的包开始组件扫描，因此所有子包（`controller`、`entity`、`mapper`、`config`、`util`）都会被自动扫描到。

```java
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
```

**导入说明**：
- `SpringApplication`：Spring Boot应用启动的核心类，提供`run()`静态方法
- `@SpringBootApplication`：Spring Boot核心注解

```java
@SpringBootApplication
public class XuezhihuiApplication {
```

**`@SpringBootApplication`**：这是一个组合注解，等价于同时标注了以下三个注解：
1. `@Configuration`：标记该类为Spring的Java配置类（可以定义`@Bean`方法）
2. `@EnableAutoConfiguration`：启用Spring Boot的自动配置机制。Spring Boot会根据classpath中的jar包自动配置项目（如检测到`spring-boot-starter-web`就自动配置Tomcat和Spring MVC；检测到`mybatis-spring-boot-starter`和`DataSource`就自动配置MyBatis）
3. `@ComponentScan`：启用组件扫描，自动扫描当前包及其子包中的`@Component`、`@Service`、`@Repository`、`@Controller`、`@Configuration`等注解并注册为Spring Bean

```java
    public static void main(String[] args) {
        SpringApplication.run(XuezhihuiApplication.class, args);
```

**`main`方法**：Java应用程序的标准入口。
- `SpringApplication.run(Class<?>, String...)`：启动Spring Boot应用的静态方法
- 第一个参数`XuezhihuiApplication.class`：告诉Spring Boot这是主配置类
- 第二个参数`args`：命令行参数，会注入到Spring环境中（可通过`@Value`或`ApplicationArguments`获取）
- 该方法执行以下操作：
  1. 创建`ApplicationContext`（Spring应用上下文）
  2. 解析`application.properties`配置文件
  3. 执行自动配置
  4. 扫描并注册所有Bean
  5. 启动内嵌Tomcat服务器（默认监听8080端口）
  6. 注册所有Controller映射

```java
        System.out.println("========================================");
        System.out.println("  学知汇·大学版 后端服务启动成功！");
        System.out.println("  访问地址: http://localhost:8080");
        System.out.println("========================================");
    }
}
```

**启动成功标志**：在控制台打印醒目的分隔线和提示信息，确认服务已正常启动并告知访问地址。

## 四、实体类 Entity 逐行解析

### 4.1 User.java（用户实体）

**文件路径**：`src/main/java/com/xuezhihui/entity/User.java`

```java
package com.xuezhihui.entity;

import lombok.Data;
```

**`import lombok.Data`**：导入Lombok的`@Data`注解。

```java
@Data
public class User {
    private Integer id;
    private String username;
    private String password;
}
```

**`@Data`**：Lombok注解，编译期自动生成以下方法：
- `getId()` / `setId(Integer)` / `getUsername()` / `setUsername(String)` / `getPassword()` / `setPassword(String)`
- `equals(Object)` / `hashCode()`（基于所有字段）
- `toString()`（包含所有字段的字符串表示）
- 无参构造器（`@NoArgsConstructor`的效果）

**字段详解**：

| 字段 | Java类型 | 数据库列名 | 数据库类型 | 业务含义 |
|------|----------|-----------|-----------|----------|
| `id` | `Integer` | `id` | `INT AUTO_INCREMENT` | 用户唯一标识，由数据库自增生成，注册时不需要赋值 |
| `username` | `String` | `username` | `VARCHAR(50) NOT NULL` | 用户名，登录和注册的凭据之一，有唯一索引约束 |
| `password` | `String` | `password` | `VARCHAR(100) NOT NULL` | 密码，明文存储（登录时与数据库直接比较） |

**字段映射机制**：MyBatis通过配置`mapUnderscoreToCamelCase=true`（在`MyBatisConfig`和`application.properties`中均配置），实现数据库下划线命名（如`user_name`）到Java驼峰命名（如`username`）的自动转换。由于`username`和`password`本身就是全小写单词，不受此映射影响。

## 五、Mapper接口与XML映射 逐行解析

### 5.1 UserMapper.java + UserMapper.xml

#### 5.1.1 Mapper接口

**文件路径**：`src/main/java/com/xuezhihui/mapper/UserMapper.java`

```java
package com.xuezhihui.mapper;

import com.xuezhihui.entity.User;
import org.apache.ibatis.annotations.Param;

public interface UserMapper {
    int insert(User user);
    User selectByUsername(@Param("username") String username);
}
```

**`public interface UserMapper`**：
- MyBatis的Mapper必须定义为接口（interface），不能是类
- MyBatis在运行时会使用JDK动态代理为该接口生成代理实现对象
- `@MapperScan("com.xuezhihui.mapper")`会扫描此包下所有接口并注册为Spring Bean

**`int insert(User user)`**：
- **功能**：插入一条新的用户记录
- **参数**：`User`对象（包含`username`和`password`字段，`id`为null由数据库自增生成）
- **返回值**：`int`表示受影响的行数，正常为1（插入1行成功）
- **SQL映射**：对应`UserMapper.xml`中`id="insert"`的`<insert>`语句

**`User selectByUsername(@Param("username") String username)`**：
- **功能**：根据用户名精确查询用户记录
- **参数**：`@Param("username")`指定MyBatis SQL中的参数名为`#{username}`，`String username`为方法入参
- **返回值**：`User`对象，如果用户名不存在则返回`null`
- **调用场景**：
  1. 注册时检查用户名是否已存在（唯一性校验）
  2. 登录时按用户名查询用户信息（密码验证）

#### 5.1.2 UserMapper.xml SQL映射

**文件路径**：`src/main/resources/mybatis/UserMapper.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
```

**XML声明和DTD**：
- 声明XML版本1.0、编码UTF-8
- DTD（文档类型定义）指定MyBatis Mapper 3.0规范，IDE可根据此DTD提供代码提示和校验

```xml
<mapper namespace="com.xuezhihui.mapper.UserMapper">
```

```java
        if (resource == null) return ResponseEntity.notFound().build();
```

**查资源记录**：按ID查询资源，如果不存在返回HTTP 404。

```java

```

**读取文件**：
- `new File(resource.getFilePath()).toPath()`：将文件路径字符串转为`Path`对象
- `Files.readAllBytes(Path)`：JDK 7+ NIO.2 API，将整个文件内容读入一个`byte[]`数组并返回
- 此方式将整个文件加载到内存，适合中小文件（<100MB）。对于超大文件，应使用`InputStream`流式输出

```java
        String encodedFilename = URLEncoder.encode(resource.getFileName(), "UTF-8").replace("+", "%20");
```

**URL编码文件名**：
- `URLEncoder.encode(文件名, "UTF-8")`：对中文文件名进行百分号编码。例如"高等数学笔记.pdf"编码为`%E9%AB%98%E7%AD%89%E6%95%B0%E5%AD%A6%E7%AC%94%E8%AE%B0.pdf`
- `.replace("+", "%20")`：`URLEncoder`会将空格编码为`+`，但RFC 5987标准要求使用`%20`，所以做一个替换
- 这是为了正确设置HTTP `Content-Disposition`头，使浏览器能正确解析中文文件名

```java

                .header(HttpHeaders.CONTENT_DISPOSITION,
                        "attachment; filename=\"" + encodedFilename + "\"; filename*=UTF-8''" + encodedFilename)
                .body(bytes);
    }
```

**构建下载响应**：
- `ResponseEntity.ok()`：创建HTTP 200响应构建器
- `.header(HttpHeaders.CONTENT_DISPOSITION, ...)`：设置Content-Disposition响应头
  - `attachment`：告诉浏览器以附件方式下载（而非在浏览器中直接打开）
  - `filename="..."`：传统格式的文件名（兼容旧浏览器）
  - `filename*=UTF-8''...`：RFC 5987标准格式（现代浏览器优先使用），`UTF-8''`是字符集和语言标签
  - 双格式提供最大化兼容性
- `.body(bytes)`：将文件字节数组作为响应体
- 浏览器收到此响应后，根据Content-Disposition头触发文件下载对话框

### 7.1 WebConfig.java（CORS跨域配置）

**文件路径**：`src/main/java/com/xuezhihui/config/WebConfig.java`

```java
package com.xuezhihui.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.CorsRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer {
```

**`@Configuration`**：标记此类为Spring配置类（等价于XML配置中的`<beans>`），Spring会在此类中查找`@Bean`方法或配置回调。

**`implements WebMvcConfigurer`**：实现Spring MVC配置接口，可以覆盖默认的MVC配置（如CORS、拦截器、视图解析器等）。

```java
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOriginPatterns("*")
                .allowedMethods("*")
                .allowedHeaders("*")
                .exposedHeaders("Content-Disposition")
                .allowCredentials(true);
    }
}
```

**`addCorsMappings`方法——逐配置项解析**：

**`addMapping("/**")`**：对所有接口路径生效。`/**`是Ant风格路径通配符，匹配所有URL路径和子路径。

**`allowedOriginPatterns("*")`**：允许所有来源域名的跨域请求。使用`allowedOriginPatterns`（而非`allowedOrigins`）是因为需要与`allowCredentials(true)`配合使用，`allowedOriginPatterns`支持通配符模式匹配。

**`allowedMethods("*")`**：允许所有HTTP方法（GET、POST、PUT、DELETE、OPTIONS等）。

**`allowedHeaders("*")`**：允许请求携带所有HTTP头（如Content-Type、Authorization等），不做限制。

**`exposedHeaders("Content-Disposition")`**：将`Content-Disposition`响应头暴露给前端JavaScript。默认情况下，浏览器CORS只暴露6个简单响应头（Cache-Control、Content-Language、Content-Type、Expires、Last-Modified、Pragma）。`Content-Disposition`不在默认列表中，所以需要显式暴露，否则前端的`response.headers.get('Content-Disposition')`将返回null（文件下载时解析文件名会失败）。

**`allowCredentials(true)`**：允许跨域请求携带Cookie和HTTP认证信息。

**为什么需要CORS配置**：前端页面可能通过`file://`协议打开或部署在不同端口，与后端`http://localhost:8080`不同源。浏览器的同源策略会阻止跨域请求，CORS配置告诉浏览器"来自其他来源的请求也是被允许的"。

**文件路径**：`src/main/java/com/xuezhihui/config/FileConfig.java`

```java
package com.xuezhihui.config;

import jakarta.servlet.MultipartConfigElement;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class FileConfig {

    @Value("${spring.servlet.multipart.max-file-size:100MB}")
    private String maxFileSize;

    @Value("${spring.servlet.multipart.max-request-size:100MB}")
    private String maxRequestSize;
```

**`@Value`注解**：
- `${spring.servlet.multipart.max-file-size:100MB}`：从`application.properties`中读取配置项的值
- `:100MB`是默认值：如果配置文件中找不到该属性，使用100MB作为默认值
- `maxFileSize`：单个上传文件的最大大小限制，100MB = 104857600字节
- `maxRequestSize`：单次HTTP请求的最大大小限制（可能包含多个文件和表单字段），100MB

```java
    @Bean
    public MultipartConfigElement multipartConfigElement() {
        String location = null;
        long maxFileSizeBytes = 100L * 1024 * 1024;      // 100MB
        long maxRequestSizeBytes = 100L * 1024 * 1024;   // 100MB
        int fileSizeThreshold = 0;

        return new MultipartConfigElement(location, maxFileSizeBytes,
                maxRequestSizeBytes, fileSizeThreshold);
    }
}
```

**`@Bean`**：将方法返回的对象注册为Spring容器中的Bean。

**`MultipartConfigElement`**：Jakarta EE标准的文件上传配置类（注意使用`jakarta.servlet`包，而非旧版`javax.servlet`）。

**构造参数解析**：
1. `location = null`：文件上传的临时存储目录。`null`表示使用Servlet容器的默认临时目录（通常是系统临时目录）
2. `maxFileSizeBytes = 100L * 1024 * 1024`：单个文件最大100MB。`100L`中的`L`后缀表示long类型，避免int溢出
3. `maxRequestSizeBytes = 100L * 1024 * 1024`：单次请求最大100MB
4. `fileSizeThreshold = 0`：文件写入磁盘的阈值。0表示所有上传的文件**先写入临时目录**再处理，避免大文件占用内存

**补充说明**：实际上Spring Boot的自动配置会从`application.properties`读取`spring.servlet.multipart.*`并自动配置，这个显式的Bean配置起到了补充和确保配置生效的作用。

### 7.4 AsyncConfig.java（异步任务配置）

**文件路径**：`src/main/java/com/xuezhihui/config/AsyncConfig.java`

```java
package com.xuezhihui.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.EnableAsync;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;

import java.util.concurrent.Executor;
import java.util.concurrent.ThreadPoolExecutor;

@Configuration
@EnableAsync
public class AsyncConfig {
```

**`@EnableAsync`**：开启Spring的异步任务支持。标注此注解后，任何使用`@Async`注解的方法都会在独立的线程中异步执行，调用方无需等待方法执行完成。适用于发送邮件、日志记录、文件处理等耗时操作。

```java
    @Bean("taskExecutor")
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
```

**`@Bean("taskExecutor")`**：
- Bean名称为`"taskExecutor"`，这是Spring异步任务的默认执行器名称
- `@Async`注解的方法默认使用名为`taskExecutor`的`Executor`执行
- 返回类型是`Executor`接口（而非具体实现类`ThreadPoolTaskExecutor`），遵循面向接口编程原则

```java
        executor.setCorePoolSize(5);
```

**`setCorePoolSize(5)`**：核心线程数5。线程池中始终保持活跃的线程数量（即使它们处于空闲状态）。

```java
        executor.setMaxPoolSize(10);
```

**`setMaxPoolSize(10)`**：最大线程数10。当核心线程都在忙且队列已满时，线程池最多可以扩展到10个线程。

```java
        executor.setQueueCapacity(100);
```

**`setQueueCapacity(100)`**：缓冲队列容量100。当核心线程都在忙时，新任务在队列中等待。队列满后才会创建新线程（最多到maxPoolSize）。

**线程池工作流程**：
1. 新任务到达 → 分配给核心线程（最多5个）
2. 5个核心线程都在忙 → 新任务进入队列（最多100个）
3. 队列满 → 创建新线程处理（最多再创建5个，总计10个）
4. 10个线程都在忙且队列满 → 执行拒绝策略

```java
        executor.setThreadNamePrefix("xuezhihui-async-");
```

**`setThreadNamePrefix`**：线程名前缀。创建的线程名称格式为`xuezhihui-async-1`、`xuezhihui-async-2`等，便于在日志和调试工具中识别异步任务线程。

```java
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
```

**`CallerRunsPolicy`**：拒绝策略——当线程池和队列都满时，任务由调用线程（即提交任务的线程，通常是主线程或HTTP请求处理线程）直接执行。这种方式不会丢失任务，同时能自然减缓新任务的提交速度（因为调用线程自己执行任务时无法提交新任务）。

```java
        executor.setWaitForTasksToCompleteOnShutdown(true);
        executor.setAwaitTerminationSeconds(30);
```

**优雅停机配置**：
- `setWaitForTasksToCompleteOnShutdown(true)`：应用关闭时等待所有正在执行的任务完成，而非强制中断
- `setAwaitTerminationSeconds(30)`：最多等待30秒。超过此时间未完成的任务将被中断

```java
        executor.initialize();
        return executor;
    }
}
```

**`initialize()`**：初始化线程池，预创建核心线程。

**`return executor`**：将线程池执行器注册为Spring Bean，供`@Async`注解使用。

**Lombok注解导入**：

- `@NoArgsConstructor`：生成无参构造器（`new ResultUtil()`）
- `@AllArgsConstructor`：生成全参构造器（`new ResultUtil(200, "操作成功", data)`）

```java

@AllArgsConstructor

```

**`<T>`泛型参数**：使`ResultUtil`可以包装任意类型的数据。例如：
- `ResultUtil<User>`：data字段类型为User
- `ResultUtil<List<Resource>>`：data字段类型为资源列表
- `ResultUtil<String>`：data字段类型为普通字符串（如"注册成功"）

```java
    private String message;
    private T data;
```

**三个核心字段**：

| 字段 | Java类型 | JSON类型 | 含义 |
|------|----------|---------|------|
| `code` | `Integer` | `number` | 状态码：`200`=成功，`500`=失败。前端通过此字段判断请求结果 |
| `message` | `String` | `string` | 提示信息：成功时为"操作成功"，失败时为具体错误描述（如"用户名已存在"） |
| `data` | `T`（泛型） | 任意 | 响应数据载荷：成功时包含具体数据，失败时为`null` |

```java
        return new ResultUtil<>(200, "操作成功", data);
    }
```

**`success(T data)`静态工厂方法**：
- 泛型方法`<T>`：方法的泛型参数独立于类的泛型参数
- 参数`T data`：要返回的数据对象
- 创建`ResultUtil`实例：code=200（成功码），message="操作成功"（固定文案），data=传入的数据
- 使用`new ResultUtil<>(...)`，钻石操作符`<>`自动推断泛型类型
- 这是所有成功响应的统一构建方式

```java

        return new ResultUtil<>(500, message, null);
    }
}
```

**`error(String message)`静态工厂方法**：
- 参数`String message`：错误描述，由各Controller的业务逻辑提供（如"用户名已存在"）
- 创建`ResultUtil`实例：code=500（错误码），message=传入的错误描述，data=null

## 十、测试类 XuezhihuiApplicationTests.java 解析

**文件路径**：`src/test/java/com/xuezhihui/XuezhihuiApplicationTests.java`

```java
package com.xuezhihui;

import org.junit.jupiter.api.Test;
import org.springframework.boot.test.context.SpringBootTest;

@SpringBootTest
class XuezhihuiApplicationTests {

    @Test
    void contextLoads() {
        // 如果上下文加载失败，此测试将抛出异常
    }
}
```

**`@SpringBootTest`**：Spring Boot集成测试注解，会加载完整的Spring应用上下文（包括所有Bean的创建和依赖注入）。与`SpringApplication.run()`功能等价，只是运行在测试环境中。

**`@Test`**：JUnit 5的测试方法注解（`org.junit.jupiter.api.Test`，注意是JUnit 5的Jupiter API，而非JUnit 4）。

**`contextLoads()`**：冒烟测试（Smoke Test）——验证Spring应用上下文能否正常加载。测试方法体为空，因为测试的是Bean的装配过程而非具体业务逻辑。如果任何一个Bean创建失败（如数据库连接失败、Mapper扫描失败、循环依赖等），`@SpringBootTest`初始化上下文的阶段就会抛出异常，测试不通过。这是Spring Boot项目中最基本也是最重要的测试。
